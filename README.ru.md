# примеры на квитанциях (receipt-driven)

[English](README.md) · **Русский**

Маленькие примеры «скопируй меня» паттерна **на квитанциях** (он же *безлоговый*, logless) от
[TimeLayer](https://timelayer-os.com).

## Идея

Обычный способ записать, что нечто произошло, — **дописать строку в лог после действия**. Такой
лог растёт бесконечно, стоит денег на хранение и — что хуже всего — любой, у кого есть доступ,
может отредактировать его задним числом. Вы доверяете хранителю лога.

Подход на квитанциях переворачивает это: вы **требуете подписанную квитанцию *до* действия** и
храните квитанцию вместо строки лога.

Квитанция — это самодостаточная пара файлов:

```
cert.tlcert      сертификат
bundle.tlbundle  подтверждающие данные
```

подписанная **кворумом независимых операторов на Ed25519** (ни одна сторона — включая
TimeLayer — не может выписать её в одиночку). Проверяется **офлайн**, без сети и без сервера
ключей, с помощью open-source верификатора:

```bash
timelayer-verifier verify cert.tlcert bundle.tlbundle
# -> VALID FINAL    (exit 0)  подлинна и полна
# -> UNVERIFIABLE   (exit 1)  отказ
```

Нечему расти, нечего подделать, не за что нам доверять — доказательство путешествует вместе с
действием, и любой может перепроверить его.

## Правило: fail-closed

> Нет валидной квитанции — нет действия.

Любое сомнение — нет верификатора, ненулевой код выхода, неожиданный вывод — считается «невалидно»,
и действие **не** выполняется.

## Запустить пример

Нужен open-source верификатор (одно скачивание) и Rust.

```bash
./run.sh
```

`run.sh` скачивает верификатор под вашу ОС, собирает пример и запускает его дважды: один раз
против вложенной **валидной** квитанции (действие выполняется) и один раз против **подменённой**
копии (действие отклоняется). Сама проверка полностью офлайн.

## Что здесь

| Путь | Что |
|------|------|
| `rust/src/main.rs` | Шаблон. `receipt_is_valid()` — переиспользуемый гейт, оставьте его; замените `do_the_real_work()` реальным эффектом вашей программы. |
| `sample-receipt/`  | Настоящая пара квитанции (`cert.tlcert` + `bundle.tlbundle`), которую можно проверить офлайн. |
| `run.sh`           | Скачивает верификатор и прогоняет валидный + подменённый случаи. |

## Откуда берутся квитанции

Эти примеры *потребляют и проверяют* квитанции. Квитанции **выпускает** сеть TimeLayer (см.
[TL-Agent](https://github.com/TimeLayer-OS/TL-Agent) — SDK агента, который запрашивает квитанции
разрешения и выпускает квитанции провенанса). Исходники верификатора — в
[timelayer-verifier](https://github.com/TimeLayer-OS/timelayer-verifier): прочитайте, соберите —
и вы не зависите ни от одного бинарника, который не скомпилировали сами.
